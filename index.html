Know More: Regular Expressions

A Regular Expression (Regex) is a formal way to describe patterns in text. Think of it as a set of rules that defines a "language" of valid strings mathematically.

Core Operations

  • Concatenation (ab): Strings must follow one another sequentially. E.g., ab matches exactly "ab".
  • Union (a+b): Represents the logical "OR". E.g., a+b matches either "a" or "b".
  • Kleene Star (*): Specifies that the preceding element can occur zero or more times indefinitely. E.g., a* matches the empty string "ε", "a", "aa", etc.
  • Optional (?): The preceding element can occur zero or one time. E.g., a? is computationally equivalent to (a+ε).

Under the Hood

Our compiler builds a Non-deterministic Finite Automaton (NFA) from your regex using Thompson's construction logic. The NFA allows multiple branching paths and "free" Epsilon jumps. It then transforms this complex web into an optimized Deterministic Finite Automaton (DFA) via Subset Construction—giving every character uniquely defined transitions.
